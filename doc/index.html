A compact self-contained realtime audio-synthesis engine

Lyd is meant to be used as an audio engine in games for realtime effects as well as background music. It is also a testbed for the original author to experiment with audio synthesis and various forms of composition.

Features

  • library/engine
    • Small binary footprint
    • language binding friendly C API
    • Custom audio synthesis language with vm/compiler
    • Audio output drivers for: SDL, ao, jack
  • audio daemon
    • MIDI/OSC control

API documentation

Not much yet, the entire public API is defined in lyd.h, and simple examples of it's use can be seen in some small examples.

Lyd binary / audio daemon

In addition to installing a library and public headers, lyd installs a commandline audio synthesis daemon that can be operated via MIDI or OSC (depending on library availability at compile time.

The OSC support needs rethinking, and probably re-integration with the midi support. Allowing repatching instruments used with MIDI on the fly from OSC.

The midi support creates a virtual ALSA MIDI device that USB MIDI keyboards can be hooked up to as well as other pieces of midi software like rosegarden or the commandline player pmidi. qjackctl provides a way to connect ALSA midi devices (it does this without needing the jackd server to be started).

pmidi -p 128:0 music.mid

Roadmap/plans/todo/bugs (contributions welcome)

  • Create instruments presets for all of general MIDI.
  • Synthesize longer segments for each voice (storing arrays of return values for each op in the virtual machine). This should reduce the overhead allowing more simultanous and complex voices with lower CPU overhead.
  • Integrated midi parser and playback API.
  • Loading PCM data from memory for using wav samples
  • The compiler contains a leak.
  • Audio output driver using the safe ALSA subset.
